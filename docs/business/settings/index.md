# Settings

Ottu gives merchants full control over their payment platform through a centralized settings dashboard. From branding and security to webhooks and reporting, every aspect of your payment system can be configured to match your business requirements.

## Settings Sections

| Section | What It Covers |
|---------|---------------|
| **[Global Configuration](./global.md)** | Merchant identity, branding, contact details, security options, and system-wide behavior |
| **[Webhooks Configuration](./webhooks.md)** | Real-time event notifications, retry logic, SSL verification, and plugin-specific webhook routing |
| **[Transaction Reports](./transaction-reports.md)** | Periodic and flexible transaction reports, delivery via email or SFTP, and custom report fields |
| **[API Keys](./api-keys.mdx)** | Generating, managing, and revoking API keys for secure integration |
| **[URL Shortener](./url-shortener.md)** | Shortening public payment links for easier sharing |

## Where to Start

If you are setting up Ottu for the first time, start with **[Global Configuration](./global.md)** to establish your merchant identity, branding, and foundational system behavior. From there:

- Set up **[Webhooks](./webhooks.md)** to receive real-time payment and operation notifications on your server.
- Configure **[Transaction Reports](./transaction-reports.md)** to automate financial reporting for your team.
- Generate **[API Keys](./api-keys.mdx)** if your developers need to integrate with the [Ottu API](/developers/).
- Enable the **[URL Shortener](./url-shortener.md)** to create concise payment links for customers.
